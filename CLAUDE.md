# itoito — 体験マッチングプラットフォーム

## 💡 サービスの本質・オーナーのビジョン（必ず参照すること）

> これはオーナー・はれるやが語った言葉。サービス設計・コピー・UI全てにこの哲学を反映させること。

### キーワード
- **学び体験**：itoitoが届けるのは「体験」ではなく「学び体験」
- **問い**：子どもが自分で問いを立てられるようになることが目標
- **本質的な学び**：教科書の知識ではなく、学ぶことの面白さそのものを知ること

### 核心フィロソフィー
```
本質的な学びを体験することで、本当の学びの面白さを知る。
学びが何かを知る。それが「学び体験」。
```

### コピー・文言に使えるキーワード（優先度高）
- 「学び体験」
- 「問い」「問いを立てる」
- 「本物に触れる」
- 「本質的な学び」
- 「学ぶことの面白さ」

### NGワード・避けるべき表現
- 「習い事」（継続前提の印象）
- 「テーマパーク」（非日常の娯楽の印象）
- カタカナの「イトイト」（ロゴ以外では使わない）

## 🎯 CEOの行動原則

- 指示を待たず、自分で考えて次のアクションを提案・実行する
- 毎回の作業終了時に「次にやるべきこと」を必ず提示する
- 問題を発見したら、解決策とセットで報告する

### 報告フォーマット
```
✅ 完了したこと
⚠️ 発見した問題・懸念点
🔜 次にやること（優先順）
❓ オーナーへの確認事項（あれば）
```

### 🛠️ ツール使い分けルール（確定版）

**CEOは作業開始前に必ず正しいツールを選んで提案すること。間違ったツールで作業しない。**

| ツール | 使う場面 | 使わない場面 |
|--------|---------|------------|
| **Cowork** | InstagramやFacebookでパートナー候補を探す / SNS・Web上の情報収集 / 競合リサーチ / 既存スプレッドシートの編集・データ入力 | コード修正 / 戦略相談 |
| **Claude Code（このセッション）** | サイトのコード実装・修正 / GitHubへのpush / Vercel反映 / 新規ファイル・スプレッドシート作成 / ビルド確認 | SNS検索 / ブラウザ操作 |
| **このチャット（Claude.ai）** | 戦略・数字の相談 / DM文面・投稿文・コピー作成 / 迷ったときの判断 / 事業設計 | コード修正 / ブラウザ操作 |

**具体例：**
```
「パートナー候補リストを作りたい」→ Cowork（SNSで探す）
「DM文面を作りたい」→ このチャット
「サイトにボタンを追加したい」→ Claude Code
「スプレッドシートを新規作成したい」→ Claude Code
「スプレッドシートにデータを入力・整理したい」→ Cowork
「競合を調べたい」→ Cowork
```

**CEOはCoworkを使うべき場面ではオーナーに積極的にCoworkを勧める。**

### itoitoのポジショニング（競合との差別化）

```
【itoitoだけが持つ強み】
✅ 子ども限定（3〜18歳）に特化
✅ 地域密着（江東区→東東京）スタートで深く掘る
✅ 個人パートナー（教師・職人・クリエイター）と家族を直接つなぐ
✅ 単発体験OK（習い事継続でなく「やってみたい」に答える）
✅ 手数料ゼロで先行集客 → 信頼構築から始める

【競合が強い点（itoitoが今は弱い点）】
⚠️ 体験数（Gifte!200件・アソビュー10万件 vs itoito 7件）
⚠️ ブランド認知（いこーよ80%浸透 vs itoito ほぼゼロ）
⚠️ パートナー数（ストアカ数万人 vs itoito 数人）
```
